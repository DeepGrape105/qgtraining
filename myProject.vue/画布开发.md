# 画布开发

过从零到⼀构建⼀个MVP（MinimumViableProduct，;小可行产品）级别的画布应用，让你综合运⽤前后端知识，深⼊理解图形编辑器的核⼼原理与⼯程挑战。你将直⾯图形渲染、交互设计、性能优化、数据持久化等真实场景，并有机会探索协同编辑等前沿领域。交付⼀个稳定、流畅、可解释的作品，并在过程中锻炼架构设计、技术取舍与复杂问题拆解的能⼒。

**本地 Undo/Redo (Command 模式)**：在 `src/core/history.js` 中维护一个堆栈，记录操作 `Diff`。用户按下 `Ctrl+Z` 时立即更新内存中的 `elements`，不产生网络请求。

**远端 Snapshot (持久化)**：用户点击“保存”时，调用 `PUT /api/canvas/:id`。后端将数据持久化至 MySQL，并创建一个不可变的快照。

## 架构选择

| **层级**     | **名称**                   | **在你项目中的具体角色**                                     |
| ------------ | -------------------------- | ------------------------------------------------------------ |
| **顶层架构** | **数据驱动 (Data-Driven)** | 核心灵魂。定义了 elements 数组作为唯一真相，所有模块都绕着它转。 |
| **组织模式** | **模块化 (Modular)**       | 骨架。把 Canvas 绘图逻辑、UI  面板逻辑、数据处理逻辑拆分到不同文件。 |
| **功能模式** | **命令模式 (Command)**     | 技能插件。专门负责处理“撤销/重做”，它是对数据修改动作的二次封装。 |

## 对于第三方库的约束

• 禁⽌使⽤：严禁使⽤任何⾼度封装的图形库或框架，如 react-flow、 tldraw 、 konva 、 fabric.js 等。

• 允许使⽤：

基础渲染库：允许使⽤ Pixi.js 等底层图形渲染库，辅助你更⾼效地与WebGL/Canvas API交互。

局部能⼒库：在⾮核⼼业务逻辑上，可使⽤成熟的三⽅库解决特定问题（例如，使⽤ prosemirror 或slate.js 实现富⽂本编辑）。

##核心元素数据结构 (Element Schema)

所有存入 `elements` 数组的对象必须严格遵守以下通用属性，并根据类型增加特定属性：

###通用基础属性 (所有元素共有)

- `id` (String): 唯一标识符（建议格式：`type_timestamp_random`）。
- `type` (String): 元素类型，枚举值：`rect` | `circle` | `image` | `text`。
- `x` (Number): 元素左上角相对于画布的 X 坐标。
- `y` (Number): 元素左上角相对于画布的 Y 坐标。
- `width` (Number): 宽度。
- `height` (Number): 高度。
- `fill` (String): 填充颜色（十六进制，如 `#FFFFFF`）。
- `stroke` (String): 边框颜色。
- `strokeWidth` (Number): 边框粗细。
- `opacity` (Number): 透明度 (0-1)。
- `zIndex` (Number): 层级顺序（数值越大越靠前）。
- `rotation` (Number): 旋转角度（0-360）。

###特定类型属性

- **图片类 (type: 'image')**:
  - `url` (String): 后端返回的相对路径（需通过 `getFullAssetUrl` 转换）。
- **文本类 (type: 'text')**:
  - `content` (String): 富文本的 HTML 字符串。

### `Toolbar.vue` (顶栏：指令发送者)

**职责：** 切换当前的“工作模式”（是想画图，还是想选中移动）。

- **HTML 结构：** 一排按钮，每个按钮对应一个图标（如矩形、圆形、鼠标箭头、清空图标）。
- **实现逻辑：**
  - 点击按钮时，调用 Store 里的 `setMode(mode)` 函数。
  - 例如：点击“矩形”按钮 -> `canvasStore.currentMode = 'rect'`。
- **代码亮点：** 可以通过 CSS 给当前选中的模式按钮加个高亮边框，提升用户体验。

### `CanvasBoard.vue` (画布：舞台中心)

**职责：** 整个应用最复杂的地方，负责**渲染**和**监听原始鼠标事件**。

- **HTML 结构：** * 一个 `<canvas>` 标签。
  - 外层包裹一个 `div`，用于控制画布在屏幕居中或滚动。
- **实现逻辑：**
  - **挂载阶段 (onMounted)**：获取 Canvas 实例，调用 `useCanvas.js` 里的初始化函数。
  - **事件监听**：在 `<canvas>` 上绑定 `@mousedown`, `@mousemove`, `@mouseup`。
  - **事件转发**：将这些原生的 DOM 事件，传给 `useInteraction.js` 里的处理函数。
- **Canvas 高清适配**：在 `CanvasBoard.vue` 中，记得根据 `window.devicePixelRatio` 设置 Canvas 的 `width` 属性（物理像素）和 CSS `style.width`（逻辑像素）

### `PropertyPanel.vue` (属性面板：数据编辑器)

**职责：** 当用户选中某个图形时，展示并修改它的细节（颜色、坐标、宽高）。

- **HTML 结构：** * 一组输入框 (`<input type="number">`)。
  - 一个颜色选择器 (`<input type="color">`)。
  - 一个删除按钮。
- **实现逻辑：**
  - **条件渲染 (v-if)**：如果没有选中任何元素，显示“请先选择一个图形”；如果选中了，才显示编辑界面。
  - **双向绑定 (v-model)**：直接绑定 Store 中“当前选中元素”的属性。
  - **例子**：`<input v-model="activeElement.fill">`。用户改颜色，数据立刻变，由于 `useCanvas` 在监听数据，画布也会立刻重绘。

- **状态定义：**
  - `elements`: 响应式数组，存储所有图形 JSON。
  - `selection`: 记录当前选中的元素 ID。
  - `canvasConfig`: 画布宽高、背景色等。
- **函数封装：**
  - `getElementById(id)`: 快速查找元素。
  - `updateActiveElement(props)`: 修改当前选中的属性（颜色、宽高）。

 ### `src/composables/useElements.js` (图形数据管理)

**逻辑：** 所有的图形操作（添加、删除、更新属性）都封装在这里。

- **函数封装：**
  - `addElement(type)`: 根据类型生成默认 JSON 对象并 push 到数组。
  - `updateElement(id, props)`: 修改指定元素的属性。
  - `removeElement(id)`: 过滤掉指定 ID 的元素。
- **健壮性考量：** 每个元素必须有唯一的 `id，`使用 `nanoid` 库

### `src/composables/useCanvas.js` (渲染引擎)

- **职责：** 将 JSON 数据转化为像素。
  - **函数封装：**
    - `initCanvas(canvasEl)`: 设置上下文，处理 **DevicePixelRatio（高分屏适配）** 避免模糊。
    - `clear()`: 每一帧开始前清空画布。
    - `drawElement(el)`: 核心分支逻辑。通过 `switch(el.type)` 调用底层的 `ctx.rect` 或 `ctx.arc`。
    - `renderLoop()`: 使用 `requestAnimationFrame` 驱动的循环函数。
  - **逻辑说明：** 这里要实现“**离屏渲染**”或“**按需重绘**”以保证 100+ 元素的流畅（P0 要求）。
  - **逻辑：** 不同的图形对应不同的 Canvas 原生路径 API。
    - **矩形 (rect)**：使用 `ctx.rect(x, y, w, h)`。
    - **圆形 (circle)**：使用 `ctx.arc(x, y, radius, 0, Math.PI * 2)`。
    - **三角形 (triangle)**：
      - **实现：** 需要通过 `ctx.moveTo(x1, y1)`、`ctx.lineTo(x2, y2)` 手动连点。
    - **通用样式设置：**
      - **背景色：** `ctx.fillStyle = color` + `ctx.fill()`。
      - **边框宽度：** `ctx.lineWidth = width`。
      - **边框颜色：** `ctx.strokeStyle = color` + `ctx.stroke()`。

 ### `src/composables/useInteraction.js` (交互处理器)

- **职责：** 监听鼠标事件并翻译为业务动作。
  - **函数封装：**
    - `handleMouseDown(e)`:
      1. **转换逻辑：** - 1. 获取 `rect = canvas.getBoundingClientRect()`。
         - 1. 计算 `x = (clientX - rect.left) * (canvas.width / rect.width)`。
         - 1. 计算 `y = (clientY - rect.top) * (canvas.height / rect.height)`。
         - *注：利用画布物理像素与 CSS 像素的比例进行缩放，确保在 4K 屏下点击位置依然精准。*
      2. 调用 `math.js` 里的函数进行“拾取测试”。
      3. 确定选中目标，记录按下时的初始位置（用于计算偏移）。
    - `handleMouseMove(e)`:
      - 如果处于拖拽状态，计算当前鼠标位置与初始位置的 $deltaX$ 和 $deltaY$，更新 `store` 里元素的 `x` 和 `y`。
    - `handleMouseUp()`: 结束状态，清除监听。
  - **逻辑说明：** 采用“**状态机**”思想（Idle -> Selecting -> Dragging）
- **健壮性考量：** 拖拽时要配合 `window.addEventListener('mousemove')`，确保鼠标移出画布时依然能正常结束动作。
- **坐标映射逻辑**：
  - 鼠标事件触发时，禁止直接使用 `clientX/Y`。
  - **转换公式**：
    1. 通过 `canvas.getBoundingClientRect()` 获取画布在屏幕上的 CSS 边界。
    2. 计算鼠标相对于画布左上角的 CSS 偏移量。
    3. 乘以 **缩放比例系数**（即 `canvas.width / rect.width`），以适配 `devicePixelRatio` 引起的高清缩放。
- **交互频率限制**：
  - 在 `handleMouseMove` 中，仅更新内存坐标数据，具体的“重绘”动作由 `requestAnimationFrame` 驱动的渲染循环统一执行。

###  `src/utils/math.js` (数学工具箱)

**职责：** 处理 Canvas 里的物理规则。这是手写图形引擎的“灵魂”。

- **函数封装：**
  - `isPointInRect(x, y, rect)`: 判断点是否在矩形内。
  - `isPointInCircle(x, y, circle)`: 利用勾股定理 $d = \sqrt{(x_1-x_2)^2 + (y_1-y_2)^2}$ 判断点是否在圆内。
  - `getDistance(p1, p2)`: 计算两点距离。
- **逻辑说明：** 这里是**碰撞检测（Hit Testing）**的核心。当用户点击画布，你会调用这些函数来确定“点中了谁”。

###`src/utils/storage.js` (持久化工具)

**职责：** 满足 P0 级的保存需求。

- **函数封装：**
  - `saveToLocal(data)`: 封装 `localStorage.setItem`。
  - `loadFromLocal()`: 恢复数据，并包含**错误处理逻辑**（防止 JSON 格式损坏导致崩溃）。

### `src/utils/imageProcessor.js` (滤镜工厂)

**职责：** 这个文件不关心 Vue，也不关心 Canvas 的坐标，它只处理**原始数据（ImageData）**。

- 核心原理：Canvas 的 `getImageData` 会返回一个超长的数组 `[r1, g1, b1, a1, r2, g2, b2, a2, ...]`。这个文件就是通过循环这个数组来修改颜色。


- **applyGrayscale(imageData) (灰度)：**
  - **逻辑：** 遍历每个像素，取 RGB 的平均值或加权平均值。
  - **公式：** `Luma = R*0.299 + G*0.587 + B*0.114`。
  - **操作：** 将 R、G、B 全部设为这个 `Luma` 值。
- **applyBrightness(imageData, value) (亮度)：**
  - **逻辑：** 给每个 R、G、B 通道加上一个增量 `value`（范围 -255 到 255）。
  - **注意：** 需要用 `Math.min(255, Math.max(0, val))` 防止数值溢出。
- **applyContrast(imageData, value) (对比度)：**
  - **逻辑：** 调整颜色偏离 128（中间灰）的程度。
  - **公式：** `f = (259*(value+255)) / (255*(259-value))`，然后 `newColor = f*(oldColor-128)+128`。

### `src/core/Renderer.js` (绘图工厂)

**职责：** 这是你的**“指令翻译官”**。它负责把 JSON 里的属性（如 `width: 100`）转换成 Canvas 的 API 调用（如 `ctx.rect(...)`）

**负责的函数逻辑：**

- **draw(ctx, element) (入口函数)：**
  - 负责保存状态 `ctx.save()`。
  - 设置通用样式（全局透明度 `globalAlpha`、阴影 `shadowBlur`）。
  - 根据 `element.type` 路由到具体的子函数。
  - 最后恢复状态 `ctx.restore()`。
- **drawRect(ctx, element) / drawCircle(ctx, element)：**
  - 处理边框逻辑：先 `fill()` 填充，再 `stroke()` 勾边。
  - 处理边框宽度 `ctx.lineWidth`。
- **drawText(ctx, element)：**
  - 调用 `ctx.font` 设置字号字体。
  - 处理文本对齐。
- **drawImage(ctx, element)：**
  - 如果元素有滤镜参数，先在离屏 Canvas 处理像素，再画到主画布。
- **图片加载逻辑：**
  - 在 Canvas 绘制 `type: 'image'` 的元素前，必须先调用 `getFullAssetUrl(element.url)` 转换路径。
  - 示例：`img.src = getFullAssetUrl(element.url)`。
- **资产预加载机制 (Asset Cache)**：
  - 禁止在每一帧渲染循环中创建 `new Image()`。
  - 必须实现一个 `AssetLoader` 类或模块，维护一个 `Map` 对象作为缓存。
  - **逻辑流程**：当数据改变时，先通过 URL 加载图片并存入 `Map`；渲染循环只负责从 `Map` 中读取已 `onload` 的 HTMLImageElement 进行绘制。
- **富文本渲染约束 (ForeignObject)**：
  - 输入的 HTML 必须经过 **Inline CSS 处理**（将所有样式写在标签的 `style` 属性内）。
  - 渲染为 SVG 前，必须检查并注入必要的 Base64 字体编码，否则 Canvas 无法渲染非系统字体。

### `src/hooks/usePersistence.js` (持久化与防丢逻辑)

**封装函数：useLocalDraft(elements)**:

- **实现逻辑：**
  1. **监听：** 使用 `watch(elements, ..., { deep: true })`。
  2. **防抖 (Debounce)：** 数据变化后不立即写磁盘，而是等待 500ms-1s，防止高频重绘时拖慢性能。
  3. **写入：** 将 JSON 字符串存入 `localStorage`，键名为 `canvas_draft_[id]`。
- **状态标记：** 可以在 UI 上显示一个“草稿已保存到本地”的微小提示。

### `src/hooks/useVersionControl.js` (版本切换逻辑)

**职责：** 处理前端 Canvas 渲染引擎与后端快照数据的对接。

- **实现逻辑：**

- **封装函数：**
  - `undoToLastSave()`: 撤销到最近的一次“保存点”（放弃所有本地未保存修改）。

###需要多个调用函数实现的功能

####富文本渲染

**内联样式化**：前端 ProseMirror 输出 HTML 时，使用插件将 CSS 类名转换为 `style="..."` 内联属性。

**字体处理**：如果使用非系统字体，在生成 SVG 字符串前，必须在 `<svg>` 的 `<style>` 标签中插入 `@font-face` 的 **Base64** 编码，否则 `foreignObject` 无法解析网络字体。

#####`components/CanvasBoard.vue` (负责：编辑状态切换)

由于 ProseMirror 是一个实实在在的 DOM 节点，它必须挂载在 HTML 上。因此，控制“编辑框什么时候出现、出现在哪”的逻辑在 `CanvasBoard.vue` 中。

- **函数名：** `handleDoubleClick(e)`
- **实现逻辑：**
  1. 监听 Canvas 的双击事件。
  2. 调用 `useInteraction.js` 判定是否点中了“文本元素”。
  3. 如果点中，计算该元素在屏幕上的绝对位置（`x`, `y`, `width`, `height`）。
  4. 显示一个绝对定位的 `div`（编辑器容器），覆盖在 Canvas 上方，并初始化 ProseMirror 实例。

##### `src/core/Renderer.js` (负责：渲染状态 - 绘制)

当用户完成编辑（失去焦点或点击外部），我们需要把文字画回 Canvas。

- **函数名：** `drawRichText(ctx, element)`

- **实现逻辑：**

  1. **输入：** 接收 `element.content`（ProseMirror 输出的 HTML 字符串）。

  2. **构造 SVG：** 将 HTML 嵌入到 `foreignObject` 模板中。

  3. **转为图片：** ```javascriptconst data = ` <svg xmlns="http://www.w3.org/2000/svg" width="${w}" height="${h}"> <foreignObject width="100%" height="100%"> <div xmlns="http://www.w3.org/1999/xhtml">${element.content}</div> </foreignObject> </svg>`;const img = new Image();img.src = 'data:image/svg+xml;base64,' + window.btoa(unescape(encodeURIComponent(data)));

  4. **策略：** 建立 **“离屏缓存”**。

     **逻辑：** - 图片/SVG 转换完成后，将其存储在 `Map` 缓存中。

     - `drawRichText` 仅负责从缓存读取已加载的图像并 `drawImage`。
     - **禁止**在渲染循环中直接进行 `new Image()` 或 `btoa` 转换操作。

  5. **渲染：** `ctx.drawImage(img, element.x, element.y)`。

- **图片加载逻辑：**

  - 在 Canvas 绘制 `type: 'image'` 的元素前，必须先调用 `getFullAssetUrl(element.url)` 转换路径。
  - 示例：`img.src = getFullAssetUrl(element.url)`。

- **资产预加载机制 (Asset Cache)**：

  - 禁止在每一帧渲染循环中创建 `new Image()`。
  - 必须实现一个 `AssetLoader` 类或模块，维护一个 `Map` 对象作为缓存。
  - **逻辑流程**：当数据改变时，先通过 URL 加载图片并存入 `Map`；渲染循环只负责从 `Map` 中读取已 `onload` 的 HTMLImageElement 进行绘制。

- **富文本渲染约束 (ForeignObject)**：

  - 输入的 HTML 必须经过 **Inline CSS 处理**（将所有样式写在标签的 `style` 属性内）。
  - 渲染为 SVG 前，必须检查并注入必要的 Base64 字体编码，否则 Canvas 无法渲染非系统字体。

##### `src/composables/useElements.js` (负责：数据同步)

这个文件负责确保 ProseMirror 里的内容能存进你的 JSON 大脑里。

- **函数名：** `updateTextContent(id, newHTML)`
- **实现逻辑：**
  1. 当 ProseMirror 内容改变时，调用此函数。
  2. 找到对应 ID 的元素，更新其 `content` 属性。
  3. 触发全局响应式更新，通知 Canvas 重新渲染。

### `services/canvasService.js` (业务逻辑核心 - **最重要**)

- **负责功能：** 处理“保存”动作引发的连锁反应。
- **封装函数：**
  - 1. **saveAndSnapshot(canvasId, elements, config)**
       - **输入参数**：`canvasId` (String), `elements` (Array), `config` (Object)。
       - **逻辑步骤**：
         1. 从连接池获取连接：`const conn = await pool.getConnection()`。
         2. **开启事务**：`await conn.beginTransaction()`。
         3. **序列化**：将 `elements` 和 `config` 分别进行 `JSON.stringify()` 转为字符串。
         4. **更新主表**：执行 `UPDATE canvases SET content = ?, config = ? WHERE id = ?`。
         5. **创建快照**：执行 `INSERT INTO snapshots (canvas_id, content) VALUES (?, ?)`，存储当前这一刻的全量 JSON。
         6. **提交/回滚**：成功则 `await conn.commit()`，捕获到任何错误则 `await conn.rollback()` 并抛出异常。
         7. **释放连接**：`conn.release()`。
  - **rollbackToSnapshot(canvasId, snapshotId)**
    - **逻辑步骤**：
      1. 从 `snapshots` 表查询 `content` 和 `canvas_id`。
      2. 校验 `snapshot` 是否属于该 `canvasId`。
      3. 执行 `UPDATE canvases SET content = ? WHERE id = ?`，将历史数据覆盖回主表。

### `controllers/assetController.js` (资源控制器)

**负责功能：** 处理多媒体资源的物理上传、类型过滤、大小限制，并返回可供前端渲染器（Renderer.js）直接使用的相对路径。

- **封装函数：** `handleUpload(req, res)`

- **实现逻辑：**

  1. **存储配置 (Multer Storage)：**
     - 使用 `multer.diskStorage` 将文件保存至项目根目录的 `uploads/` 文件夹。
     - **文件名重命名：** 采用 `Date.now() + '-' + Math.round(Math.random() * 1E9)` 配合原扩展名，确保文件名唯一，防止覆盖。
  2. **文件过滤器 (File Filter)：**
     - 严格校验 `file.mimetype`。仅允许 `['image/png', 'image/jpeg', 'image/jpg']`。
     - 若类型不符，返回 `400 Bad Request` 及错误信息。
  3. **限制条件 (Limits)：**
     - 设定 `fileSize: 5 * 1024 * 1024` (5MB)。若超限，由 Multer 抛出错误并在全局错误中间件中捕获。
  4. **返回逻辑 (关键)：**
     - 仅返回以 `/uploads/` 开头的相对路径，由前端 `request.js` 中的 `getFullAssetUrl` 函数进行动态拼接。

- **返回 JSON 示例：**

  ~~~
  {

    "data": {
      "url": "/uploads/1712476000123-my-image.png",
      "mimetype": "image/png",
      "size": 102400
    },

## 逻辑需求闭环

**前端操作时：** 仅触发前端 `usePersistence.js` 中的 `localStorage` 写入，**后端不参与**，保证流畅度。

**点击“保存”时：**

- 前端发送 `PUT /api/canvas/1`（带 Token）。
- 后端 `canvasController` 接收，`canvasService` 同时更新主记录并创建一个新快照。
- 后端返回 `200 OK` 及最新版本 ID。
- 前端收到成功后，执行 `localStorage.clear()`，清空本地暂存。

**点击“撤销到上个保存版本”时：**

- 前端询问用户是否丢弃未保存的修改。
- 前端请求 `GET /api/canvas/1`。
- 后端返回主表中最后一次“保存”的内容。
- 前端用该内容覆盖 `elements` 数组，Canvas 触发重绘。
